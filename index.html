Pendahuluan

Modul praktikum ini menguraikan perancangan sistem backend RESTful API Manajemen Data Panen Petani berbasis runtime environment Node.js. Implementasi ini berfungsi sebagai pembuktian praktis dari implementasi sepuluh pilar arsitektur RESTful global seperti yang diilustrasikan pada acuan skema rancangan ekosistem perkuliahan.

Prinsip RESTful API Penerapan Spesifik pada Project Panen Petani
Penamaan Resource Mengisolasi objek data menggunakan nomina jamak: /api/v1/panen
Operasi HTTP (CRUD) GET POST PUT PATCH DELETE
Control Versioning Menghindari tabrakan rute struktural dengan menyisipkan URL prefix path: /api/v1/
Content Negotiation Proses tukar-menukar data payload klien dan server seutuhnya dikunci dalam format standar JSON
Penyaringan Query Membatasi data dinamis menggunakan pencarian kata kunci filter URL parameter: ?jenis_tanaman=...
Status Code Standar Representasi indikator status terklasifikasi: 200 (OK), 201 (Created), 401 (Unauthorized), 404 (Not Found)
Keamanan Gerbang Validasi hak akses masuk terproteksi wajib melampirkan komponen header kustom: X-API-Key
Sifat Idempotent Method GET, PUT, dan DELETE didesain konsisten agar tidak merusak data jika dipanggil berulang kali.
HATEOAS Navigasi Menyematkan struktur hypermedia tautan referensi otomatis di dalam properti objek respons berkode _links
Prinsip Stateless Server tidak menyimpan histori memori session; setiap request diproses secara independen dan utuh.

Persiapan Environment

A. Mengaktifkan Local Server

Inisialisasi dependensi library node_modules dan eksekusi development server melalui perintah terminal berikut:

# Menjalankan skrip otomatis nodemon development

npm run dev

Setelah perintah di atas dieksekusi, terminal akan melakukan kompilasi berkas utama. Pastikan log terminal menampilkan status sukses berjalan aktif seperti pada screenshot di bawah ini:

Gambar: Tampilan log sirkulasi kompilasi backend server berjalan aktif pada port 3000.

B. Alternatif Pengujian Eksternal: Thunder Client

Jika Anda memilih untuk tidak menggunakan aplikasi pihak ketiga eksternal seperti Postman, Anda bisa memanfaatkan ekstensi **Thunder Client** yang terintegrasi langsung di dalam lingkungan kerja VS Code. Ekstensi ini jauh lebih ringan dan efisien untuk kebutuhan pengujian RESTful API secara lokal.

Cara Instalasi Thunder Client di VS Code:

  1. Buka aplikasi VS Code Anda, lalu akses menu Extensions di sidebar kiri (Ctrl+Shift+X).
  2. Pada kolom pencarian, ketik kunci pencarian Thunder Client.
  3. Klik tombol Install pada baris ekstensi resmi berlambang petir ungu.
  4. Setelah berhasil terpasang, ikon petir Thunder Client akan muncul secara otomatis di bilah menu sidebar kiri VS Code Anda dan siap digunakan sebagai REST Client.

5. Lembar Kerja Praktik Pengujian API (Thunder Client)

P1

Akses Root Endpoint API

HTTP 200 OK

Penjelasan & Arti Query Status: Respons mengembalikan status 200 OK. Pengujian rute utama dilakukan tanpa melampirkan token keamanan sebagai penanda gerbang awal bahwa server aplikasi backend Express berjalan normal di http://localhost:3000. Rute ini turut memuat hypermedia links (_links) menuju rute dokumentasi.

GET http://localhost:3000/

{
  "success": true,
  "message": "👋 Selamat datang di RESTful API Panen Petani!",
  "version": "1.0.0"
}
P2

Eksplorasi Dokumentasi API Terintegrasi

HTTP 200 OK

Penjelasan & Arti Query Status: Respons mengembalikan status 200 OK. Menyediakan informasi skema parameter, rute yang tersedia, serta instruksi autentikasi header agar pengembang aplikasi frontend mengetahui kunci rahasia yang wajib dikirimkan saat mengakses data panen.

GET http://localhost:3000/api/docs

{
  "success": true,
  "message": "Dokumentasi API Panen Petani",
  "apiKey": { "header": "X-API-Key", "value": "tif-api-key-panen-2026" }
}
P3

Uji Proteksi Gerbang Keamanan (Tanpa API Key)

HTTP 401 Unauthorized

Penjelasan & Arti Query Status: Respons mengembalikan status 401 Unauthorized. Hal ini membuktikan bahwa komponen middleware apiKey.js telah bekerja secara efektif memblokir akses ilegal yang mencoba mengambil data panen tanpa menyertakan kredensial valid pada komponen header.

GET http://localhost:3000/api/v1/panen

{
  "success": false,
  "error": { "code": 401, "message": "API Key diperlukan. Kirim header: X-API-Key" }
}
P4

Ambil Semua Records Data Panen

HTTP 200 OK

Penjelasan & Arti Query Status: Respons mengembalikan status 200 OK ketika pengujian dilakukan dengan menyisipkan X-API-Key: tif-api-key-panen-2026 pada HTTP Headers. Server Express berhasil melakukan operasi pembacaan berkas (*Read All*) dan menyajikan data seluruh komoditas dalam bentuk array of objects.

GET http://localhost:3000/api/v1/panen
Headers: X-API-Key: tif-api-key-panen-2026
P5

Penyaringan Data (Query Parameter)

HTTP 200 OK

Penjelasan & Arti Query Status: Respons mengembalikan status 200 OK. Backend menangkap nilai string filter melalui variable req.query.jenis_tanaman=Padi untuk memisahkan komoditas padi secara spesifik dan memangkas ukuran pengiriman data payload ke client side.

GET http://localhost:3000/api/v1/panen?jenis_tanaman=Padi
P6

Ambil Data Spesifik Berdasarkan ID Panen

HTTP 200 OK

Penjelasan & Arti Query Status: Respons mengembalikan status **200 OK**. Menggunakan segmen parameter URL (req.params.id_panen) untuk mengisolasi record id tunggal. Data dikembalikan secara presisi mencakup properti berat, kualitas, harga jual, dan total pendapatan terhitung.

GET http://localhost:3000/api/v1/panen/401

{
  "success": true,
  "message": "Data panen ditemukan",
  "data": {
    "id_panen": 401,
    "jenis_tanaman": "Padi Varietas Unggul"
  }
}
P7

Tambah Pencatatan Panen Baru

HTTP 201 Created

Penjelasan & Arti Query Status: Respons mengembalikan status **201 Created**. Menandakan operasi penulisan resource baru (*Create Operation*) via kata kerja HTTP POST sukses disubmit, diproses oleh fungsi bisnis, dan berhasil disimpan ke dalam daftar memori penyimpanan.

POST http://localhost:3000/api/v1/panen
Request Body: { "id_petani": 3, "jenis_tanaman": "Kelapa", "berat_kg": 1000 }

{
  "success": true,
  "message": "Data panen berhasil ditambahkan"
}
P8

Modifikasi Objek Menyeluruh (PUT Full Update)

HTTP 200 OK

Penjelasan & Arti Query Status: Respons mengembalikan status 200 OK. Sesuai prinsip RESTful, kata kerja PUT bertindak menggantikan/menimpa keseluruhan struktur data lama dengan payload baru.

PUT http://localhost:3000/api/v1/panen/403
Payload JSON Body: { "jenis_tanaman": "Kacang Panjang", "berat_kg": 1000 }
P9

Modifikasi Parsial Data (PATCH Partial Update)

HTTP 200 OK

Penjelasan & Arti Query Status: Respons mengembalikan status 200 OK. Pengujian menggunakan kata kerja PATCH membuktikan efisiensi performa backend karena modifikasi parsial hanya merubah properti jenis_tanaman menjadi "Cabai Rawit" tanpa mengganggu nilai properti lain di sekitarnya.

PATCH http://localhost:3000/api/v1/panen/403
P10

Operasi Penghapusan Record Berkas Panen

HTTP 200 OK

Penjelasan & Arti Query Status: Respons mengembalikan status 200 OK. Eksekusi metode destruktif DELETE berhasil menemukan target identifikasi resource (id_panen: 408) dan mencabut entitas tersebut secara permanen dari basis data lokal proyek.

DELETE http://localhost:3000/api/v1/panen/408

{
  "success": true,
  "message": "Data panen berhasil dihapus"
}
P11

Uji Penanganan Error (Endpoint 404 Malformed)

HTTP 404 Not Found

Penjelasan & Arti Query Status: Respons mengembalikan status **404 Not Found**. Kasus ini terjadi akibat client melakukan penembakan request ke alamat rute yang keliru (/api/v1/tanaman). Skrip fallback menangkap rute liar ini dan mengembalikan pesan petunjuk (*hint*) kustom yang rapi.

DELETE http://localhost:3000/api/v1/tanaman

{
  "success": false,
  "error": {
    "code": 404,
    "message": "Endpoint DELETE /api/v1/tanaman tidak ditemukan"
  }
}
P12

Pengujian Eksternal Portabel Menggunakan Terminal cURL

HTTP 200 OK

Penjelasan & Arti Query Status: Mengembalikan status 200 OK langsung pada layar antarmuka CLI terminal konsol. Langkah penutup ini membuktikan validitas fungsionalitas RESTful API kita yang bersifat independen, portabel, dan siap dikonsumsi oleh berbagai client platform global.

curl -X GET http://localhost:3000/api/v1/panen -H "X-API-Key: tif-api-key-panen-2026"
